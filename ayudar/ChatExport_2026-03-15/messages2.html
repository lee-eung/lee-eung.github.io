1. direct output
2. strong direct source
3. direct source 중 related/약한 후보
4. message/thread artifact expansion
5. 그 밖의 약한 파생 힌트

즉 이제 자동 매핑 결과를 볼 때:

• output 계열은 더 강하게
• expanded artifact 계열은 더 조심스럽게
보는 기준을 명시했어.

───

신뢰 등급 해석

High trust

• direct output
• 해석 가능한 강한 direct source
• 충돌 신호 없음

→ retrieval / answer prep에 직접 활용 가능

Medium trust

• related_to
• 중간 confidence
• artifact expansion 기반

→ 보조 맥락으로 사용

Low trust

• 짧고 모호한 메시지
• 여러 파생 hop에 의존
• corroboration 부족

→ 내부 후보 수준으로만 두고, 답변 본문에서는 단정하지 않음

───

불확실성 처리 규칙

이번에 명확히 고정한 것:

• 저장된 링크는 재사용 가능한 근거 신호이지 자동 확정 진실이 아님
• expanded layer는 standalone proof처럼 말하지 않음
• direct layer와 expanded layer가 다르면 direct 우선
• 정책/자격/행정/기한/컴플라이언스에 영향 큰 질문은 자동 매핑 결과만으로 확정형 문장을 쓰지 않음

즉 “자동 매핑상 연결이 있다”와
“확정 근거다”를 분리하는 규칙을 넣은 거야.

이 wrapper가 하는 일

즉 새 세션에서 그냥 무작정 깊게 파는 게 아니라,

Step 1. Quick pass

• recall bundle 먼저 보고
• 최근 메시지 / 최근 산출물 / 최근 answer events만으로 충분하면 여기서 멈춤

Step 2. Context pass

• reply-context bundle 보고
• linked docs / outputs / 구조가 충분히 드러나면 여기서 멈춤

Step 3. Deeper retrieval fallback

아래일 때만 deeper retrieval로 내려감:

• 자동 매핑이 related/expanded 위주
• high-stakes 질문
• 충돌/최신성 문제
• 공식 확인 필요

이렇게 stop/deepen 기준까지 같이 명시돼.
